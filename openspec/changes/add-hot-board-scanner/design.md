## Context

热榜股票筛选是一个多维度分析系统，需要整合多种数据源：
1. **龙虎榜数据**：机构/游资买卖明细（akshare）
2. **行情数据**：涨幅、量比、换手率（akshare/tushare）
3. **板块数据**：板块涨幅、涨停数（akshare）
4. **技术指标**：均线、突破形态（已有能力）

约束：
- 依赖 akshare 的免费接口，可能存在限流
- 数据实时性要求较高，建议收盘后使用
- 游资名单需要定期更新

## Goals / Non-Goals

**Goals:**
- 实现多榜单数据获取和整合
- 实现机构和游资识别
- 实现多维度评分模型
- 提供清晰的CLI输出

**Non-Goals:**
- 不做实时推送（非实时系统）
- 不做自动化交易
- 不做回测框架（可后续扩展）

## Decisions

### 1. 数据源选择
- **决策**：使用 akshare 作为主要数据源
- **原因**：
  - akshare 龙虎榜接口丰富
  - 免费无需token
  - 接口稳定
- **备选**：tushare（需要积分）

### 2. 评分模型设计
- **决策**：采用加权评分模型，总分100分
- **权重分配**：
  - 多榜共振 40分（最重要）
  - 资金性质 30分
  - 技术形态 20分
  - 板块效应 10分
- **原因**：热点和资金是短线核心，技术和板块为辅助

### 3. 游资名单管理
- **决策**：硬编码知名游资名单，支持配置文件扩展
- **原因**：
  - 游资名单相对稳定
  - 配置文件方便用户自定义
- **位置**：`config.yaml` 中的 `hot_board.famous_zy` 字段

### 4. 模块结构
```
pichip/hot_board/
├── __init__.py
├── fetcher.py      # 数据获取
├── scoring.py      # 评分引擎
├── filters.py      # 风险过滤
└── display.py      # 输出格式化
```

## Risks / Trade-offs

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| akshare接口限流 | 无法获取数据 | 添加重试机制，提示用户稍后再试 |
| 游资名单过时 | 识别不准确 | 支持配置文件更新，定期维护 |
| 评分模型偏差 | 筛选结果不理想 | 权重可配置，支持用户调整 |
| 数据延迟 | 实时性不足 | 明确建议收盘后使用 |

## Migration Plan

无需迁移，纯新增功能。
