# Change: Add Control Index (主力控盘指数)

## Why

当前项目专注于K线形态识别，但缺乏对主力资金控盘程度的量化分析能力。控盘指数可以帮助用户：
- 判断是否有主力资金关注
- 识别主力控盘程度（高度控盘/中度控盘/低控盘）
- 辅助决策买入时机（高控盘股更适合跟随）

## What Changes

### 新增功能
- 新增 `pichip control --code <代码>` 命令：查看单只股票的控盘指数
- 新增 `pichip control --scan` 命令：扫描全市场高控盘股票
- 实现综合控盘指数评分模型（0-100分）

### 评分模型

**控盘指数 = 0.4 × 筹码集中度 + 0.3 × 量价控盘系数 + 0.3 × 抗跌性**

#### 1. 筹码集中度（权重40%）
- 数据来源：季报股东户数、人均持股
- 评分标准：股东户数环比变化
  - 减少 >15%：90-100分（高度集中）
  - 减少 5-15%：70-89分（中度集中）
  - 减少 0-5%：50-69分（略微集中）
  - 增加 0-5%：30-49分（略微分散）
  - 增加 >5%：0-29分（明显分散）

#### 2. 量价控盘系数（权重30%）
- 数据来源：日K线数据（本地）
- 计算方法：近20日缩量上涨天数占比 + 换手率趋势
- 评分标准：
  - 缩量上涨占比 >70% + 换手率递减：90-100分（高度控盘）
  - 缩量上涨占比 50-70%：70-89分（中度控盘）
  - 缩量上涨占比 30-50%：50-69分（轻度控盘）
  - 缩量上涨占比 <30%：0-49分（控盘弱）

#### 3. 抗跌性（权重30%）
- 数据来源：日K线数据 + 大盘数据
- 计算方法：大盘下跌日个股相对表现
- 评分标准：
  - 大盘跌时个股平均上涨：90-100分（强护盘）
  - 相对抗跌（比大盘少跌>5%）：70-89分（有护盘）
  - 与大盘同步：50-69分（无护盘）
  - 比大盘跌更多：0-49分（弱势）

### 控盘等级划分

| 综合得分 | 等级 | 判断 |
|----------|------|------|
| 80-100 | 高度控盘 | 主力主导，可关注 |
| 60-79 | 中高控盘 | 有主力关注 |
| 40-59 | 中度控盘 | 主力存在但力度一般 |
| 20-39 | 低控盘 | 主力参与度低 |
| 0-19 | 无控盘 | 散户主导 |

### 输出格式

```
股票: 000001 平安银行
控盘指数: 72分 [中高控盘]

┌─────────────────────────────────────────────────┐
│ 子指标          │ 得分 │ 权重 │ 加权得分 │ 状态   │
├─────────────────────────────────────────────────┤
│ 筹码集中度      │  80  │ 40%  │   32.0   │ ↑集中  │
│ 量价控盘系数    │  65  │ 30%  │   19.5   │ 中等   │
│ 抗跌性          │  68  │ 30%  │   20.4   │ 较强   │
└─────────────────────────────────────────────────┘

解读：
• 股东户数连续2季度减少，筹码趋于集中
• 近20日缩量上涨占比55%，有一定控盘
• 相对大盘抗跌性良好，有主力护盘迹象
```

## Impact

- Affected specs: 新增 `control-index` 能力
- Affected code:
  - `pichip/cli.py` - 新增 `control` 命令
  - `pichip/data/fetcher.py` - 新增股东户数数据获取
  - `pichip/control/` - 新模块，包含控盘指数计算逻辑
- Dependencies:
  - 股东户数数据：tushare（季报数据）
  - 大盘数据：akshare 或 tushare
  - 日K数据：复用现有本地缓存
